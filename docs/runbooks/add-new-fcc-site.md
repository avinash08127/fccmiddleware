# Operational Runbook: Adding a New FCC Site

Step-by-step guide for onboarding a new fuel station with FCC adapter integration.

---

## Prerequisites

- [ ] Physical FCC device installed and connected to station LAN
- [ ] FCC vendor and model identified (DOMS, Radix, or Petronite)
- [ ] LAN IP address and port of the FCC device confirmed
- [ ] FCC credentials obtained from vendor (access code, shared secret, or OAuth credentials)
- [ ] Legal entity and site created in cloud backend / Portal
- [ ] Edge agent device provisioned and registered to the site

---

## Step 1: Create Site and Legal Entity (Cloud Portal)

1. Navigate to **Portal > Master Data > Legal Entities**
2. Create or select the legal entity for this station
3. Navigate to **Portal > Sites > Add Site**
4. Fill in:
   - Site Code (unique identifier, e.g., `SITE-DAR-001`)
   - Legal Entity (from step 2)
   - Operating Model (e.g., `Dealer`, `CODO`, `DODO`)
   - Timezone (IANA format, e.g., `Africa/Dar_es_Salaam`)
   - Currency (ISO 4217, e.g., `TZS`)

---

## Step 2: Configure FCC Connection (Cloud Portal)

Navigate to **Portal > Sites > [Site] > FCC Configuration**.

### For DOMS (TCP/JPL):
- Vendor: `DOMS`
- Connection Protocol: `TCP`
- Host Address: LAN IP of DOMS device
- Port: JPL port (typically `4001`)
- Access Code: FcLogon access code (provided by DOMS technician)
- Country Code: ISO country code
- Configured Pumps: `1,2,3,4` (comma-separated pump numbers)
- Heartbeat Interval: `30` seconds

### For Radix FDC:
- Vendor: `RADIX`
- Host Address: LAN IP of Radix FDC
- Port: Auth port P (e.g., `5000`; transaction port P+1 is auto-calculated)
- Shared Secret: SHA-1 signing password (from Radix technician)
- USN Code: Unique Station Number (from Radix setup)
- Pump Address Map: `{"1":{"pumpAddr":1,"fp":1}, "2":{"pumpAddr":2,"fp":1}}`

### For Petronite:
- Vendor: `PETRONITE`
- Host Address: Petronite bot hostname/IP
- Port: REST API port (typically `6000`)
- Client ID: OAuth2 client ID
- Client Secret: OAuth2 client secret
- Webhook Secret: Shared secret for webhook validation
- Webhook Listener Port: `8090` (or another available port)

---

## Step 3: Configure Nozzle Mappings

Navigate to **Portal > Sites > [Site] > Nozzle Mappings**.

Map each physical nozzle to its canonical representation:

| Odoo Pump | FCC Pump | Odoo Nozzle | FCC Nozzle | Product Code |
|-----------|----------|-------------|------------|--------------|
| 1 | 1 | 1 | 1 | UNL95 |
| 1 | 1 | 2 | 2 | DIESEL |
| 2 | 2 | 1 | 1 | UNL95 |

Set **Pump Number Offset** if FCC pump numbering doesn't start at 1.

---

## Step 4: Register Edge Agent Device

1. Install the edge agent on the station device:
   - **Desktop (.NET):** Run installer, configure `appsettings.json` with cloud URL
   - **Android (Kotlin):** Deploy APK, grant permissions, configure via settings screen

2. Register the device:
   ```
   POST /api/v1/devices/register
   {
     "deviceId": "<auto-generated UUID>",
     "siteCode": "SITE-DAR-001",
     "registrationToken": "<from Portal>"
   }
   ```

3. Verify registration: agent should show "Registered" status and start config polling.

---

## Step 5: Verify Connectivity

1. **Check agent status:**
   ```
   GET http://localhost:8585/api/status
   ```
   Expected: `"state": "FullyOnline"`, FCC heartbeat = true

2. **Check heartbeat in cloud:**
   - Portal > Sites > [Site] > Health shows last heartbeat timestamp

3. **Vendor-specific verification:**
   - **DOMS TCP:** Log should show `FcLogon_resp success` and pump count
   - **Radix:** Log should show `CMD_CODE=55 product read success`
   - **Petronite:** Log should show `OAuth token acquired` and `nozzle assignments loaded`

---

## Step 6: Test Transaction Flow

1. **Perform a test fuelling** on the physical pump

2. **Verify local buffer:**
   ```
   GET http://localhost:8585/api/transactions?status=PENDING
   ```
   Should show the test transaction with correct volume/amount

3. **Verify cloud upload:**
   - Portal > Sites > [Site] > Transactions shows the transaction
   - Status should progress: `PENDING` -> `UPLOADED` -> `SYNCED_TO_ODOO`

4. **Verify canonical fields:**
   - Volume in microlitres (e.g., 25L = 25,000,000)
   - Amount in minor units (e.g., $50.00 = 5000 for USD)
   - Correct pump/nozzle numbers
   - Correct product code

---

## Step 7: Test Pre-Auth (if applicable)

1. **Send pre-auth from Odoo POS:**
   ```
   POST http://localhost:8585/api/preauth
   {
     "preAuthId": "test-001",
     "siteCode": "SITE-DAR-001",
     "fccPumpNumber": 1,
     "fccNozzleNumber": 1,
     "productCode": "UNL95",
     "requestedAmountMinorUnits": 5000,
     "currency": "TZS"
   }
   ```

2. **Verify pump authorized:**
   - DOMS: Pump state changes to `Authorized`
   - Radix: ACKCODE=0 returned
   - Petronite: Order created and pump authorized (two-step)

3. **Complete test fuelling and verify correlation:**
   - Transaction should have `correlationId` matching the pre-auth

---

## Step 8: Configure Petronite Webhook (Petronite Only)

For Petronite sites, the Petronite bot must be configured to send webhooks to the edge agent:

1. **On the Petronite bot admin panel:**
   - Set webhook URL: `http://<agent-lan-ip>:8090/api/webhook/petronite`
   - Set webhook secret: same value as configured in Portal

2. **Verify webhook delivery:**
   - Complete a fuelling
   - Check agent logs for: `"Petronite webhook payload enqueued"`
   - Transaction should appear in local buffer

3. **For cloud-direct webhook (no edge agent):**
   - Set webhook URL: `https://<cloud-backend>/api/v1/ingest/petronite/webhook`
   - Set site code header: `X-Site-Code: SITE-DAR-001`

---

## Step 9: Production Checklist

- [ ] FCC heartbeat stable for > 1 hour
- [ ] At least 3 test transactions successfully synced to cloud
- [ ] Pre-auth tested if applicable
- [ ] Nozzle mappings verified for all pumps
- [ ] Currency and timezone correct in canonical transactions
- [ ] Agent auto-update URL configured
- [ ] Monitoring alerts configured for heartbeat failures
- [ ] Backup internet connectivity verified (failover)
- [ ] Physical labels applied to hardware (site code, device ID)

---

## Rollback Procedure

If issues are found after go-live:

1. **Disable FCC config** in Portal (set `enabled: false`) — agent stops polling immediately
2. **Check transaction buffer** for any stuck records: `GET /api/transactions?status=FAILED`
3. **Re-enable** after fixing the issue — agent resumes automatically
4. For Petronite: check for orphaned pre-auth orders via `GET /direct-authorize-requests/pending`
