## 4. Key Decisions

| # | Decision | Why | Impact |
|---|----------|-----|--------|
| D1 | Azure Entra ID for portal/employee auth, not Cognito | Cloud HLD specifies Azure Entra; aligns with enterprise identity | Portal uses MSAL.js; backend validates Entra JWTs |
| D2 | Device JWT is short-lived (24h), refreshed on every cloud sync | Limits blast radius of stolen token; devices sync frequently enough | Edge Agent must handle 401 → re-auth with refresh token |
| D3 | FCC API keys use HMAC signature, not plain bearer | Prevents replay attacks; Cloud HLD specifies HMAC for limited devices | FCC adapter must compute HMAC; cloud validates signature + timestamp |
| D4 | SQLCipher deferred to post-MVP | Room DB on single-user device with encrypted credentials is sufficient for MVP | Revisit if compliance audit requires full DB encryption |
| D5 | S3 archive uses SSE-KMS (not SSE-S3) | KMS provides key rotation, audit trail, and access control | Slightly higher cost; enables per-legal-entity key policies later |
| D6 | Bootstrap token valid 72 hours, single-use | Balances field deployment logistics (shipped devices) with security | Expired tokens require portal re-generation |
| D7 | LAN API key is a 256-bit random token, not JWT | Simple shared secret sufficient for trusted LAN; no claims needed | Stored in Android Keystore on primary, distributed via QR to secondaries |

## 5. Detailed Specification

### 5.1 Device Registration Flow

**Bootstrap Token Generation**

| Property | Value |
|----------|-------|
| Generated by | Portal (System Admin or Ops Manager) or `POST /api/v1/admin/bootstrap-tokens` |
| Format | 32-byte cryptographically random, Base64URL-encoded |
| Storage | `bootstrap_tokens` table: `token_hash` (SHA-256), `site_code`, `legal_entity_id`, `created_by`, `expires_at`, `used_at` |
| Expiry | 72 hours from generation |
| Usage | Single-use; marked consumed on successful registration |
| Delivery | QR code displayed in portal, scanned by Edge Agent setup screen |

**Registration Sequence**

| Step | Actor | Action | Detail |
|------|-------|--------|--------|
| 1 | Portal user | Generate bootstrap token | Selects site, generates token, displays QR |
| 2 | Edge Agent | Scan QR | Extracts: `bootstrapToken`, `cloudBaseUrl`, `siteCode` |
| 3 | Edge Agent | `POST /api/v1/agent/register` | Body: `{ bootstrapToken, siteCode, deviceFingerprint, appVersion }` |
| 4 | Cloud | Validate | Token exists, not used, not expired, siteCode matches |
| 5 | Cloud | Create device record | Generate `deviceId` (UUID v4), store in `agent_registrations` |
| 6 | Cloud | Issue tokens | `deviceToken` (access JWT, 24h) + `refreshToken` (opaque, 90 days) |
| 7 | Cloud | Mark bootstrap token used | Set `used_at`, link to `deviceId` |
| 8 | Cloud | Return response | `{ deviceId, deviceToken, refreshToken, config }` |
| 9 | Edge Agent | Store securely | `deviceToken` + `refreshToken` in Android Keystore; `deviceId` in EncryptedSharedPreferences |

**Device JWT Claims**

```json
{
  "sub": "<deviceId>",
  "iss": "fcc-middleware-cloud",
  "aud": "fcc-middleware-api",
  "site": "<siteCode>",
  "lei": "<legalEntityId>",
  "roles": ["edge-agent"],
  "iat": 1741651200,
  "exp": 1741737600
}
```

| Claim | Type | Purpose |
|-------|------|---------|
| `sub` | UUID | Device identity; used for tenant scoping |
| `site` | string | Restricts API access to this site's data |
| `lei` | UUID | Legal entity scoping for multi-tenancy |
| `roles` | string[] | Always `["edge-agent"]` for devices |
| `exp` | unix timestamp | 24 hours from issuance |

**Token Refresh**

| Property | Value |
|----------|-------|
| Endpoint | `POST /api/v1/agent/token/refresh` |
| Request | `{ refreshToken }` |
| Response | `{ deviceToken, refreshToken }` (new refresh token issued — rotation) |
| Refresh token lifetime | 90 days |
| Refresh token storage | `device_refresh_tokens` table: `token_hash`, `device_id`, `expires_at`, `revoked_at` |
| On 401 from any API | Edge Agent calls refresh endpoint; if refresh also fails → enter re-registration state |
| Rotation | Each refresh issues a new refresh token and invalidates the old one |

**Decommission Flow**

| Step | Actor | Action |
|------|-------|--------|
| 1 | Portal user | Select device → "Decommission" |
| 2 | Cloud | Set `agent_registrations.status = DECOMMISSIONED`, `decommissioned_at = now()` |
| 3 | Cloud | Revoke all refresh tokens for device |
| 4 | Cloud | Next API call from device returns `403 DEVICE_DECOMMISSIONED` |
| 5 | Edge Agent | On 403 with this code: stop all sync, show "Device decommissioned" on diagnostics screen |

### 5.2 Cloud API Authentication

| Caller | Endpoint Pattern | Auth Method | Credential Source | Validation |
|--------|-----------------|-------------|-------------------|------------|
| FCC (modern) | `POST /ingest` | mTLS client cert | Per-FCC cert provisioned via portal | Cert CN matches registered `fccId`; cert not revoked |
| FCC (limited) | `POST /ingest` | API Key + HMAC | `X-Api-Key` header + `X-Signature` header | Key exists, active, HMAC valid, timestamp within ±5 min |
| Odoo | `GET /transactions`, `POST /acknowledge` | API Key | `X-Api-Key` header | Key exists, active, scoped to `legalEntityId` |
| Edge Agent | All `/agent/*`, `/upload`, `/preauth`, `/synced-status` | JWT Bearer | `Authorization: Bearer <deviceToken>` | Signature valid, not expired, device not decommissioned |
| Edge Agent (registration) | `POST /agent/register` | Bootstrap Token | In request body | Token valid, unused, unexpired |
| Portal (admin) | `All /admin/*` | Azure Entra JWT | `Authorization: Bearer <entraToken>` | Entra signature valid, audience matches, roles extracted |
| Databricks | `POST /master-data/*` | API Key | `X-Api-Key` header | Key exists, active, role = `master-data-sync` |

**FCC API Key Lifecycle**

| Operation | How | Detail |
|-----------|-----|--------|
| Generation | Portal → `POST /api/v1/admin/fcc/{fccId}/api-key` | 256-bit random, returned once, stored as SHA-256 hash + encrypted value in Secrets Manager |
| HMAC signing | FCC computes `HMAC-SHA256(apiKey, method + path + timestamp + bodyHash)` | Cloud recomputes and compares |
| Rotation | Generate new key → grace period (both keys valid for 24h) → old key auto-expires | `fcc_api_keys` table tracks `active_from`, `active_until` |
| Revocation | Portal → deactivate key → immediate effect | Set `revoked_at`; 403 on next request |

**Odoo API Key Lifecycle**

| Property | Value |
|----------|-------|
| Scope | Per legal entity |
| Generation | Portal (System Admin) or seeded during legal entity onboarding |
| Storage | SHA-256 hash in `odoo_api_keys` table; plaintext in AWS Secrets Manager for Odoo team |
| Rotation | Same grace-period mechanism as FCC keys |
| No HMAC | Odoo calls are server-to-server over TLS; plain API key is sufficient |

**Databricks / Master Data Sync Key**

| Property | Value |
|----------|-------|
| Scope | Global (one key for all master data operations) |
| Permissions | Write-only to master data tables; no transaction access |
| Rotation | Manual; coordinated with Databricks pipeline config |
| Alternative | IAM role-based auth if both services run in same AWS account — evaluate during infra setup |

### 5.3 Edge Agent Security

**Android Keystore Usage**

| Key Alias | Algorithm | Purpose | Rotation |
|-----------|-----------|---------|----------|
| `fcc-middleware-device-jwt` | N/A (stores opaque token) | Stores current `deviceToken` JWT | Replaced on every token refresh |
| `fcc-middleware-refresh-token` | N/A (stores opaque token) | Stores current `refreshToken` | Replaced on every token refresh |
| `fcc-middleware-fcc-cred` | AES-256-GCM | Encrypts FCC credentials (password/cert) at rest | On FCC credential change via config push |
| `fcc-middleware-lan-key` | N/A (stores opaque token) | Stores LAN API key for multi-HHT access | On explicit rotation via portal |

All keys are hardware-backed where device supports it (Urovo i9100 TEE). Keys are non-exportable, user-authentication-not-required (service runs unattended).

**EncryptedSharedPreferences**

| Key | Value Stored | Why Encrypted |
|-----|-------------|---------------|
| `device_id` | UUID | Device identity — prevents spoofing if device storage accessed |
| `site_code` | string | Site binding — tampering could cause cross-site data leakage |
| `legal_entity_id` | UUID | Tenant binding |
| `cloud_base_url` | string | Prevents redirect attacks |
| `fcc_host` | string | FCC network address |
| `fcc_port` | int | FCC connection port |

Non-sensitive config (poll intervals, batch sizes, log level, buffer settings) stored in regular SharedPreferences — no encryption needed.

**LAN API Key**

| Property | Value |
|----------|-------|
| Format | 256-bit random, Base64URL-encoded |
| Generated by | Primary HHT during initial registration (cloud returns it as part of config) |
| Storage on primary | Android Keystore (`fcc-middleware-lan-key`) |
| Distribution | QR code displayed on primary's diagnostics screen; scanned by non-primary HHTs |
| Validation | Non-primary sends `X-Api-Key` header; primary compares constant-time against stored key |
| Rotation | Portal triggers regeneration → pushed to primary via config update → non-primaries must re-scan QR |

**Certificate Pinning**

| Domain | Pin Target | Rotation Procedure |
|--------|-----------|-------------------|
| `*.fcc-middleware.{env}.example.com` (cloud API) | Intermediate CA public key (not leaf cert) | Add new pin via config push → 30-day overlap → remove old pin via config push |
| No pinning for FCC LAN | FCC is on local network, IP-based | N/A |

Pin configuration delivered as part of `SiteConfig` from cloud, enabling rotation without APK update. Pinning uses OkHttp `CertificatePinner`.

### 5.4 Data Protection

**Log Redaction Rules**

| Field | Where Produced | Rule |
|-------|---------------|------|
| FCC API key | Cloud ingestion middleware | Never log; replace with `[REDACTED]` |
| FCC credentials (password, cert) | Edge Agent FCC adapter, Cloud FCC config | Never log |
| Device JWT | Edge Agent HTTP client, Cloud auth middleware | Log last 8 chars only: `...aBcDeFgH` |
| Refresh token | Edge Agent, Cloud token endpoint | Never log |
| Bootstrap token | Cloud registration endpoint | Never log |
| Customer TIN (`taxIdentificationNumber`) | Cloud transaction processing, Edge Agent | Never log; mask in portal UI: `***1234` |
| Odoo API key | Cloud Odoo integration | Never log |
| LAN API key | Edge Agent local API | Never log |

Implementation: structured logging with a `SensitiveFieldFilter` that intercepts log serialization. Fields annotated with `[Sensitive]` (.NET) or `@Sensitive` (Kotlin) in models are auto-redacted.

**Encryption at Rest**

| Component | Mechanism | Key Management |
|-----------|-----------|---------------|
| Aurora PostgreSQL | AWS-managed AES-256 encryption (default, always-on) | AWS-managed key; no action needed |
| ElastiCache Redis | Encryption at rest enabled on cluster creation | AWS-managed key |
| S3 archive | SSE-KMS with default `aws/s3` KMS key | Auto-rotation enabled; per-bucket policy |
| FCC credentials in DB | Field-level AES-256-GCM | KMS customer-managed key (`fcc-middleware/fcc-credentials`) |
| Edge Agent SQLite | No encryption (MVP); SQLCipher evaluated post-MVP | N/A for MVP |
| Edge Agent sensitive config | EncryptedSharedPreferences (AES-256-SIV + AES-256-GCM) | Android Keystore master key |

**SQLCipher Decision**

| Aspect | MVP | Post-MVP |
|--------|-----|----------|
| Decision | Do not use SQLCipher | Evaluate based on compliance audit findings |
| Rationale | Single-user device, no multi-user risk; sensitive fields (FCC creds, tokens) stored outside SQLite in Keystore/EncryptedSharedPrefs; SQLCipher adds ~10% query overhead and APK size | If regulatory or customer audits require full database encryption |
| Trigger to revisit | Compliance audit request, or storing PII beyond current scope in Room DB |

## 6. Validation and Edge Cases

| Scenario | Handling |
|----------|---------|
| Bootstrap token used twice | Second registration returns `409 BOOTSTRAP_TOKEN_ALREADY_USED` |
| Bootstrap token expired | `401 BOOTSTRAP_TOKEN_EXPIRED`; portal user generates new one |
| Device JWT expired, refresh token valid | Transparent refresh; Edge Agent retries original request |
| Device JWT expired, refresh token expired | Edge Agent enters `REGISTRATION_REQUIRED` state; shows alert on diagnostics screen; requires new bootstrap token |
| Device JWT expired, refresh token revoked (decommission) | `403 DEVICE_DECOMMISSIONED`; Edge Agent stops all sync |
| FCC API key in grace period | Both old and new keys accepted; cloud logs which key was used |
| HMAC timestamp outside ±5 min window | `401 INVALID_SIGNATURE`; FCC must have NTP-synced clock |
| LAN API key missing on non-primary request | `401 UNAUTHORIZED`; no fallback |
| Certificate pin mismatch | Connection refused by OkHttp; Edge Agent logs error, retries on next sync cycle; does NOT fall back to unpinned |
